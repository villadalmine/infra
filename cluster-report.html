Deployment Roadmap

1K3s + Cilium CNI✅ deployed
  • K3s v1.35.2+k3s1 — 5 servers (embedded etcd HA) + 4 RK1 agents
  • Cilium 1.19.2 — eBPF CNI, LB-IPAM (192.168.178.200), Gateway API CRDs
⚠ make core alone = broken cluster (flannel disabled). Always run both:
make core && make networking
2TLS + DNS + GitOps✅ deployed
  • cert-manager v1.20.1 — internal CA + wildcard *.cluster.home TLS
  • Gateway API — shared HTTPRoute ingress (ClusterIP + HTTPRoute pattern — never LoadBalancer for HTTP)
  • Pi-hole — wildcard DNS at 192.168.178.203 → uses local-path (SQLite incompatible with SMB/CIFS)
  • ArgoCD — GitOps at argocd.cluster.home
make ingress && make dns && make gitops
3NAS Storage CSI✅ deployed
  • SMB/CIFS CSI driver — NAS at 192.168.178.102
  • smb-nas StorageClass — default PVC backend for observability/AI/security
  • smb-nas-pg — PostgreSQL-optimized share (kagent)
⚠ MUST run before observability/ai/security — they need smb-nas PVCs
make storage
4Observability Stack✅ deployed
  • kube-prometheus-stack 82.18.0 — Prometheus + Grafana + AlertManager
  • Grafana Tempo 1.26.7 — distributed tracing (pinned: v2.0.0 has template bug)
  • Grafana Loki 6.55.0 — log aggregation (smb-nas PVC)
  • Grafana Alloy 1.7.0 — OTel-compatible collector (replaces deprecated Grafana Agent)
make observability
5AI Stack✅ deployed
  • Docker Registry v2 — ARM64 image registry (smb-nas PVC)
  • LiteLLM proxy — universal LLM router: free → free2 (NVIDIA) → cheap (qwen-turbo)
  • Hermes Agent 0.7.0 — Telegram + web AI assistant (kaniko ARM64 build ~60min)
  • HolmesGPT 0.24.0 — AI SRE: queries K8s cluster, explains failures
  • Holmes UI — nginx:alpine chat at holmes-ui.cluster.home
  • kagent 0.8.5 — multi-tenant AI agent platform with K8s CRDs
make ai && make ai-holmes && make kagent
6Security Stack✅ deployed
  • NeuVector 5.5.0 (CNCF Sandbox) — runtime security, process whitelisting, vulnerability scanning
  • Dedicated LoadBalancer 192.168.178.204 — Cilium Gateway API doesn't support TLS passthrough
make security
