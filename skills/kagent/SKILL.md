# kagent — AI Agent Platform (K8s Operator)

CNCF sandbox project by Solo.io. Kubernetes-native AI agent orchestration with MCP server lifecycle management (kmcp).

## Architecture

```
kagent namespace
├── kagent-controller   CRD reconciler — watches all namespaces for Agent/MCPServer
├── kagent-ui           Next.js dashboard at kagent.cluster.home
├── kagent-db           PostgreSQL 18 (bundled, smb-nas PVC, pgvector ready)
└── kmcp-controller     Manages MCPServer CRDs (bundled since v0.7)

ai namespace
└── litellm-proxy       OpenAI-compatible backend → kagent uses this
```

## CRDs

| CRD | Scope | Purpose |
|-----|-------|---------|
| `Agent` | Namespaced | Defines an AI agent with system prompt, tools, and model config |
| `ModelConfig` | Namespaced | LLM provider + model config (one per namespace by default) |
| `MCPServer` | Namespaced | MCP server deployed and managed by kmcp controller |
| `RemoteMCPServer` | Namespaced | External MCP server via SSE endpoint |
| `ToolServer` | Namespaced | HTTP tool server (non-MCP) |
| `Memory` | Namespaced | Vector memory store for agents |

## Multi-tenancy model

- **Controller** (`kagent` ns): watches ALL namespaces, reconciles any Agent/MCPServer CRD
- **Admin** (you): ClusterRoleBinding to `kagent-getter-role` + `kagent-writer-role` → full cluster visibility via UI
- **Tenants** (`tenant-demo` ns): RoleBinding to `kagent-writer-role` + `kagent-getter-role` → can only manage CRDs in their own namespace
- **Agent pods**: use `kagent-agent-exec` SA → read + exec, NO create/delete K8s resources

## LiteLLM integration

kagent uses the OpenAI provider with custom `baseUrl`:
```yaml
# ModelConfig CRD (generated by Helm)
apiVersion: kagent.dev/v1alpha2
kind: ModelConfig
spec:
  provider: "OpenAI"
  model: "free"          # routed by LiteLLM → free→free2→cheap fallback
  openAI:
    baseUrl: "http://litellm-proxy.ai.svc.cluster.local:4000/v1"
```

API key is LiteLLM master key (`sk-hermes-internal`).

## Deploy / upgrade

```bash
# Deploy
make kagent
# or
ansible-playbook playbooks/bootstrap.yml -i inventory/hosts.ini --tags kagent

# Upgrade chart version
# 1. Update kagent_chart_version in roles/install-kagent/defaults/main.yml
# 2. Run make kagent
```

## Health checks

```bash
# Pods
kubectl get pods -n kagent

# CRDs installed
kubectl get crds | grep kagent.dev

# ModelConfig (LiteLLM connection)
kubectl get modelconfig -n kagent -o yaml

# Built-in agents
kubectl get agents -n kagent

# UI accessible
curl -I https://kagent.cluster.home

# Controller logs
kubectl logs -n kagent deploy/kagent-controller --tail=50

# Test LiteLLM connectivity from controller pod
kubectl exec -n kagent deploy/kagent-controller -- \
  wget -qO- http://litellm-proxy.ai.svc.cluster.local:4000/health
```

## Tenant management

```bash
# Create new tenant namespace
kubectl create ns tenant-acme
kubectl label ns tenant-acme kagent.dev/tenant=true

# Grant tenant RBAC (replace tenant-acme with namespace name)
kubectl create rolebinding kagent-tenant-writer -n tenant-acme \
  --clusterrole=kagent-writer-role --serviceaccount=tenant-acme:default
kubectl create rolebinding kagent-tenant-getter -n tenant-acme \
  --clusterrole=kagent-getter-role --serviceaccount=tenant-acme:default

# Tenant creates an agent in their namespace
kubectl apply -n tenant-acme -f - <<EOF
apiVersion: kagent.dev/v1alpha1
kind: Agent
metadata:
  name: my-agent
spec:
  description: "My custom agent"
  systemPrompt: "You are a helpful assistant..."
  modelConfig:
    model: free
EOF
```

## Creating a custom Agent

```yaml
apiVersion: kagent.dev/v1alpha1
kind: Agent
metadata:
  name: k8s-assistant
  namespace: ai
spec:
  description: "Kubernetes cluster assistant"
  systemPrompt: |
    You are an expert Kubernetes assistant for this homelab cluster.
    Cluster: single K3s node, namespace ai has Hermes + HolmesGPT + LiteLLM.
    You can read all resources and exec into pods. You CANNOT create or delete resources.
  modelConfig:
    model: free
  tools:
    - type: MCPServer
      name: kagent-tools   # built-in K8s tools
```

## kmcp — MCP server lifecycle

```bash
# Install kmcp CLI (for development)
# See: https://kagent.dev/docs/kmcp

# Deploy a community MCP server as K8s object
kubectl apply -n tenant-demo -f - <<EOF
apiVersion: k8s.mcpservers.org/v1alpha1
kind: MCPServer
metadata:
  name: my-mcp-server
spec:
  image: ghcr.io/example/my-mcp:latest
  command: ["node", "server.js"]
  port: 3000
EOF
```

## Helm values reference

Key values in `roles/install-kagent/defaults/main.yml`:

| Variable | Default | Purpose |
|----------|---------|---------|
| `kagent_chart_version` | `0.8.5` | Chart version |
| `kagent_llm_model` | `free` | LiteLLM model group |
| `kagent_llm_base_url` | LiteLLM svc URL | OpenAI-compatible endpoint |
| `kagent_db_storage_class` | `smb-nas` | PostgreSQL PVC storage class |
| `kagent_tenant_namespaces` | `[tenant-demo]` | Namespaces to provision with RBAC |
| `kagent_agents_enabled` | `true` | Toggle all built-in agents |
